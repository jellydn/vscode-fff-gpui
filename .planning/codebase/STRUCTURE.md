# Codebase Structure

**Analysis Date:** 2026-06-23

## Directory Layout

```
vscode-fff-gpui/
├── src/ # TypeScript source for the VS Code extension runtime
│   ├── commands/ # Command handlers and command-specific side effects
│   │   ├── findFiles.ts # Find-files command flow through the fff-gpui daemon
│   │   ├── grepFiles.ts # Grep command flow through the fff-gpui daemon
│   │   ├── openFiles.ts # VS Code document opening and cursor positioning
│   │   ├── resumeSearch.ts # In-memory last-search cache and resume dispatcher
│   │   └── runCustomTask.ts # Configured shell task picker and terminal runner
│   ├── client.ts # Unix socket client for the native fff-gpui daemon
│   ├── config.ts # VS Code configuration accessor for fff-gpui.socketPath
│   ├── extension.ts # Extension activation, deactivation, and command registration
│   ├── logger.ts # Lazy VS Code output-channel logger
│   └── types.ts # Daemon protocol request/response TypeScript interfaces
├── test/ # Vitest tests, including daemon client tests
├── scripts/ # Release automation scripts referenced by package.json
├── dist/ # Bundled VS Code extension output from tsup
├── .planning/codebase/ # Generated codebase map documents
├── AGENTS.md # Project instructions, commands, architecture notes, and quirks
└── package.json # VS Code manifest, contribution points, scripts, and dependencies
```

## Directory Purposes

**Project Root:**

- Purpose: Holds extension metadata, build/test configuration, documentation assets, project instructions, and generated/distribution artifacts.
- Contains: `package.json`, `AGENTS.md`, TypeScript/build/lint/test configuration files, release scripts, assets such as `logo.png`, and top-level documentation.
- Key files: `package.json`, `AGENTS.md`

**src:**

- Purpose: Contains all runtime TypeScript source compiled into the VS Code extension bundle.
- Contains: Extension entrypoint, daemon client, configuration helper, logger, protocol types, and command modules.
- Key files: `src/extension.ts`, `src/client.ts`, `src/config.ts`, `src/logger.ts`, `src/types.ts`

**src/commands:**

- Purpose: Contains one module per user-facing command or command-adjacent action.
- Contains: Search handlers, grep handlers, file-opening side effects, in-memory resume-search state, and custom task execution.
- Key files: `src/commands/findFiles.ts`, `src/commands/grepFiles.ts`, `src/commands/openFiles.ts`, `src/commands/resumeSearch.ts`, `src/commands/runCustomTask.ts`

**test:**

- Purpose: Contains Vitest tests for extension logic, especially socket-client behavior.
- Contains: Node-environment unit tests and mocks for native modules.
- Key files: `test/client.test.ts`

**scripts:**

- Purpose: Contains shell automation referenced by package scripts.
- Contains: Release workflow scripts.
- Key files: `scripts/release.sh`

**dist:**

- Purpose: Contains compiled extension bundle consumed by VS Code through the `main` field.
- Contains: Build output generated by `tsup` from `src/extension.ts`.
- Key files: `dist/extension.js`

**.planning/codebase:**

- Purpose: Stores generated architecture and structure maps for codebase understanding.
- Contains: Markdown analysis documents.
- Key files: `.planning/codebase/ARCHITECTURE.md`, `.planning/codebase/STRUCTURE.md`

## Key File Locations

**Entry Points:**

- `src/extension.ts`: Runtime extension entrypoint; exports `activate` and `deactivate`, registers all commands with `reactive-vscode`, logs lifecycle events, and delegates work to command modules.
- `package.json`: VS Code manifest entrypoint; points `main` to `./dist/extension.js`, declares activation events for all commands, and contributes command palette entries and keybindings.

**Configuration:**

- `package.json`: Declares `fff-gpui.socketPath` and `fff-gpui.customTasks` under `contributes.configuration`.
- `src/config.ts`: Reads `fff-gpui.socketPath` from VS Code workspace configuration.
- `AGENTS.md`: Documents project commands, coding style, architecture notes, testing approach, and known quirks for agents working in this repository.

**Core Logic:**

- `src/client.ts`: Core infrastructure boundary for Unix socket communication with the native `fff-gpui` daemon, including socket resolution, security verification, request/response handling, and timeout/error behavior.
- `src/commands/findFiles.ts`: Core find-files command flow; computes search path, sends `open_path` with `in_grep: false`, and opens selected files.
- `src/commands/grepFiles.ts`: Core grep command flow; computes search path, sends `open_path` with `in_grep: true`, and opens selected files.
- `src/commands/openFiles.ts`: Core editor integration; opens selected file paths and applies optional line/column selections.
- `src/commands/resumeSearch.ts`: Core in-memory resume behavior for repeating the last search kind.
- `src/commands/runCustomTask.ts`: Core custom-task behavior for configured terminal commands.
- `src/types.ts`: Core daemon protocol interfaces shared by client and command modules.
- `src/logger.ts`: Core logging helper used by lifecycle and search-cache paths.

**Testing:**

- `test/client.test.ts`: Unit tests for `src/client.ts`, with global `node:net` mocking and coverage for connect/write behavior, response parsing, daemon availability failures, timeout, and invalid JSON.
- `package.json`: Defines `npm test` as `vitest run`, `npm run typecheck` as `tsc --noEmit`, and `npm run lint` as `biome check`.

## Naming Conventions

**Files:**

- Lower camel case for TypeScript modules: `src/commands/findFiles.ts`, `src/commands/grepFiles.ts`, `src/commands/openFiles.ts`, `src/commands/resumeSearch.ts`, `src/commands/runCustomTask.ts`.
- Singular infrastructure/helper module names for shared services: `src/client.ts`, `src/config.ts`, `src/logger.ts`, `src/types.ts`.
- Extension entrypoint named after VS Code convention: `src/extension.ts`.
- Test files use `.test.ts`: `test/client.test.ts`.
- Generated codebase maps use uppercase descriptive Markdown names: `.planning/codebase/ARCHITECTURE.md`, `.planning/codebase/STRUCTURE.md`.

**Directories:**

- Runtime source is grouped under `src/`: `src/extension.ts` and related modules.
- User command modules are grouped under `src/commands/`: `src/commands/findFiles.ts`.
- Tests are grouped under `test/`: `test/client.test.ts`.
- Generated planning documentation is grouped under `.planning/codebase/`: `.planning/codebase/ARCHITECTURE.md`.

## Where to Add New Code

**New Feature:**

- Primary code: `src/commands/` for a new VS Code command handler, plus `src/extension.ts` to register it and `package.json` to contribute activation/command/keybinding metadata when user-facing.
- Tests: `test/` for Vitest coverage of pure logic or infrastructure behavior; follow the existing `test/client.test.ts` style for module mocks.

**New Component/Module:**

- Implementation: `src/` for shared extension services such as configuration, logging, daemon transport, or protocol helpers; `src/commands/` for command-specific orchestration and VS Code UI flows.

**Utilities:**

- Shared helpers: `src/` when used by multiple command modules, such as `src/client.ts`, `src/config.ts`, or `src/logger.ts`; command-only helpers should stay near their callers in `src/commands/`.
